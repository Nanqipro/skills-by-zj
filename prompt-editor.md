## 一、全书定位与基本基调

**书名：** 《人工智能通识教程》（第二版）

**核心定位：** 面向高校非计算机专业学生及普通读者的人工智能通识教材。读者无需具备数学或编程基础，但需要在读完本书后，对 AI 的原理、应用和社会影响形成理性、立体的认知。

**基本基调：**

- **可读性第一**：技术内容必须用生活化语言表达，优先用类比、故事和场景，而非定义和公式。
- **严谨不失温度**：医学、科研等专业章节保持学术准确，但行文不能枯燥。
- **有边界感**：所有章节都需清楚区分"AI能做什么"与"AI不能做什么"，避免夸大或贬低。
- **启发而非灌输**：鼓励读者带着问题阅读，带着思考离开，而不是被动接受结论。

## 三、写作风格分类规范

全书因受众和内容性质不同，分为两种主要写作风格，须严格按章节类型套用：

---

### 风格 A：叙事科普型（适用于第1—5章）

**核心特征：** 以故事驱动内容，以人物带动知识，以类比解释概念，以场景引发思考。

#### A1 开篇方式

每章开篇不使用"本章将介绍……"式的官方体，而是用一个**具体的历史时刻、生活场景或人物故事**切入：

```
✅ 正确示范（chapter_1.tex 开篇）：
1946年，世界上第一台通用电子计算机ENIAC在美国宾夕法尼亚大学诞生。
这台重达30吨的庞然大物……近八十年后的今天，你口袋里的那部智能手机，
其运算能力是ENIAC的数百万倍。

✅ 正确示范（chapter_3.tex 开篇）：
2015年，一款名叫Slack的办公协作软件开始在美国科技公司中流行……
十年之后，大二学生小林在期末考试前夜打开了手机上的AI学习助手……

❌ 错误示范：
本章将系统介绍人工智能的历史发展脉络及其在现代社会的应用现状。
```

#### A2 人物塑造

可引入虚构学生角色（小杨、小林等）作为读者代入视角，但须保持以下原则：

- 人物行为和场景须真实可信，贴近大学生日常
- 同一人物在同一章节内前后一致，不混用
- 人物只用于引出场景，不作为论证依据

#### A3 段落与节奏

- 段落长度：120—250 字之间，保持呼吸感，避免超过 300 字的巨型段落
- 每节末尾或关键转折前，使用三星分隔符制造节奏停顿：

```latex
\begin{center}
\textcolor{gray!70}{*\ \ *\ \ *}
\end{center}
\vspace{12pt}
```

- 关键概念第一次出现时，用 `\textbf{}` 加粗，之后正常行文

#### A4 本章导览写法

每章正文前设一个"本章导览"节，**无编号**，通过 `\section*{}` + `\addcontentsline` 加入目录：

```latex
\section*{本章导览：从"会说话"到"会做事"}
\addcontentsline{toc}{section}{本章导览：从"会说话"到"会做事"}
```

导览内容结构：
1. 承接上一章（一句话衔接）
2. 点出本章核心问题（2—3 个贯穿全章的核心命题，用 `\textbf{}` 加粗）
3. 说明各节路径（以"第一节……第二节……"等方式列出）
4. 以一个引发好奇的问句收尾，引导读者继续阅读

#### A5 图示格式（风格 A 专用）

```latex
\begin{figure}[H]
\centering
\includegraphics[width=0.88\textwidth]{./images_chapter_XX/X.X.png}
\caption*{\footnotesize\itshape\color{gray!50} 图X-X\quad 图片说明文字}
\end{figure}
```

- 图题使用斜体灰色（`\itshape\color{gray!50}`），与正文图题风格区分
- 浮动参数用 `[H]`（强制就地放置，适合叙事流中穿插图示）
- 占位图格式（尚未生成图片时）：

```latex
\begin{figure}[H]
\centering
\begin{tcolorbox}[width=0.88\textwidth, colback=gray!10, colframe=gray!50, boxrule=0.5pt, arc=2pt]
\centering
\vspace{35pt}
{\large\color{gray!70} [插图占位]}
\vspace{15pt}
{\small\color{gray!50} 图X-X：图片内容说明（建议风格描述）}
\vspace{25pt}
\end{tcolorbox}
\caption*{\footnotesize\itshape\color{gray!50} 图X-X\quad 图片标题}
\end{figure}
```

#### A6 脚注使用

引用外部数据、产品特性或研究结论时，用脚注标注来源：

```latex
\footnote{参见Netflix Tech Blog中关于个性化封面推荐算法与A/B测试的技术细节。}
```

### 风格 B：学术教材型（适用于第6—10章，医学/工程应用模块）

**核心特征：** 结构严谨，层次清晰，术语准确，每一小节有引导句、正文和小结，逻辑闭合。

#### B1 开篇方式

每章或每节开头，用 **1—2 句简洁的临床/工程场景句**切入，点明现实问题，再过渡到本节内容：

```
✅ 正确示范（chapter_7.tex 开篇）：
清晨的医院大厅刚刚亮起叫号屏，挂号、问诊、抽血、拍片、用药和随访
已经开始不断产生信息。过去，这些信息大多停留在纸张、胶片和医生经验
之中；如今，它们正在进入电子病历、影像系统、监护设备和健康管理平台……

✅ 正确示范（chapter_8.tex 小节）：
夜班病房里，医生刚提交一组新医嘱。屏幕上只是几行文字，实际却牵连着
药物选择、检查安排、护理要求、监测项目和执行时间……

❌ 错误示范：
本节介绍医嘱智能判别与规范评分系统的基本框架和技术原理。
```

#### B2 标题层级规范

```latex
% 一级：章
\chapter{章节标题}

% 二级：节（自动编号 9.1、9.2……）
\section{节标题}

% 三级：小节（自动编号 9.1.1……）
\subsection{小节标题}

% 四级：子小节（无编号，但有序号前缀，粗体）
\subsubsection*{\textbf{1. 子节标题}}
\subsubsection*{\textbf{2. 子节标题}}
……
\subsubsection*{\textbf{N. 小结}}   % 每个 subsection 末尾必有小结
```

**注意：** 第四级标题统一无编号（`*`），但内容中用数字前缀（1. 2. 3.）手动排序，粗体。

#### B3 图文引用方式

每个小节的**引言段落**必须包含图引用句，且图必须紧跟在引言段落后：

```
如图\ref{fig:ch9_XX}所示，……[用一句话概括图的核心信息]。
```

图示格式（风格 B 专用）：

```latex
\begin{figure}[htbp]
\centering
\begin{tcolorbox}[width=0.88\textwidth, colback=gray!10, colframe=gray!50,
  boxrule=0.5pt, arc=2pt, boxsep=2pt, left=2pt, right=2pt, top=2pt, bottom=2pt]
\centering
\includegraphics[width=\linewidth]{./images_chapter_07-08/X.X.png}
\end{tcolorbox}
\caption{\footnotesize\bfseries 图片标题（不含"图X.X"前缀，LaTeX 自动编号）}
  \label{fig:ch9_XX}
\end{figure}
```

- 浮动参数用 `[htbp]`（允许 LaTeX 自动优化位置）
- 图题使用粗体黑色（`\bfseries`）
- 必须设置 `\label`，与正文 `\ref` 配对

#### B4 列表格式

风格 B 中的列表统一使用以下格式，条目结构为"**名称：** 解释文字"：

```latex
\begin{itemize}
  \item \textbf{名称}或名称加冒号：解释内容，一般 2—4 句，说清含义、特点和用途。

  \item \textbf{名称二}：……（条目之间空一行，保持视觉间隔）
\end{itemize}
```

#### B5 小结写法

每个 `\subsection` 末尾必须有一个无编号小结（格式固定）：

```latex
\subsubsection*{\textbf{N. 小结}}

[1个段落，约80—120字]
用一句话概括本节讲了哪几个核心内容，
再用一句话说明它们之间的关系或在整体知识框架中的位置，
最后用一句话点出与下一节的衔接关系（可选）。
```

#### B6 版面控制（风格 B 章节开头必须添加）

```latex
\raggedbottom
\setlength{\textfloatsep}{10pt plus 2pt minus 2pt}
\setlength{\floatsep}{8pt plus 2pt minus 2pt}
\setlength{\intextsep}{10pt plus 2pt minus 2pt}
```

## 四、通用写作原则（两种风格均适用）

### 4.1 句子与段落

| 规则 | 说明 |
|------|------|
| 句子长度 | 优先使用短句（25字以内）；长句不超过50字，且必须有清晰的主语 |
| 段落结构 | 每段有明确的中心句，开头或结尾点明本段核心观点 |
| 衔接词 | 使用"因此""然而""随着""与此同时""不过"等，而非"首先/其次/再次/最后"堆叠 |
| 冗余表达 | 避免"在……方面""对于……而言""需要指出的是""值得注意的是"等惯用空话 |
| 被动语态 | 尽量少用，优先用主动句式；数据和事实陈述除外 |

### 4.2 术语与概念

- 专业术语**第一次出现**时，给出简洁的中文解释或类比，不得直接引入而不解释
- 英文缩写第一次出现时给出全称：`PET（正电子发射断层扫描）`
- 不造新词，不滥用"赋能""颠覆""重塑"等空洞形容词
- 数字表达：小于100的数字用汉字（三个、十次），100以上用阿拉伯数字（512层、40万例）

### 4.3 禁止事项

- ❌ 不写"本节将……"/"本章主要包括……"式的无信息量开头
- ❌ 不写"综上所述，……"/"总而言之，……"式的机械结尾
- ❌ 不使用"非常""十分""极为""相当"等程度副词堆叠
- ❌ 不写没有具体内容支撑的大判断（如"AI彻底改变了……"而不给出具体说明）
- ❌ 不在同一段落中重复使用相同的词语或句式
- ❌ 不添加与当前写作任务无关的注释、说明、总结性文档

### 4.4 数据与案例引用

- 引用具体数据时，必须有来源语境（"某研究团队"/"公开报道显示"/"监管文件要求"）
- 案例以**功能说明**为目的，不做宣传，不过度美化任何具体产品
- 教材中的数字以"数量级准确"为标准，避免将快速变化的精确数字写死

## 五、课后习题写作规范

每章末尾设"课后习题X"，题目类型以**简答题和分析题**为主，数量视章节体量为8—12题。

**写法要求：**

- 题目语言简洁，直接指向核心知识点
- 不出纯记忆性填空或选择题
- 每道题不超过35字
- 最后1—2题应有一定开放性，引导读者结合实际思考

**格式：**

```latex
\section*{课后习题X}

\begin{enumerate}
\item 题目文字。

\item 题目文字。
……
\end{enumerate}
```

**画幅规范：**
- 横版信息图：**16:10**（2400×1500 px，或更高）
- 圆环/闭环图：**4:3**
